# 校招测评练习题库

手机优先的纯静态 PWA。网页代码、处理后的题库分片和题图位于同一仓库；原始压缩包、PDF、DOC/DOCX 和临时导入文件不进入发布仓库。

线上地址：[yunzhixu620-stack.github.io/campus-question-bank-pwa](https://yunzhixu620-stack.github.io/campus-question-bank-pwa/)

开发或导题前先阅读 [PROJECT_RULES.md](PROJECT_RULES.md) 和 [SOURCE_IMPORT_AUDIT.md](SOURCE_IMPORT_AUDIT.md)。前者记录产品规则，后者记录三份原始压缩包的逐文件核对结论。

## 当前数据

- 总记录数：7138
- 可普通练习：4631
- 待复核题：2019，不会进入随机练习、每日计划或模拟考试
- 整图资料：488，截图可能含答案标记，只用于搜索、浏览和收藏
- 数据分片：149 个，启动时仅加载目录和当前流程需要的分片
- 来源清单：`data/source-import-status.json`

不能把当前数据称为“全部安全结构化完成”。原始 ZIP 文件清单已 100% 对账，主要可结构化题库已经导入；仍有 6 份唯一题库文件因答案外露、缺少解析或旧 DOC 图文锚点不可靠而无法自动进入普通练习。它们在来源清单中保留明确状态，避免错题、串图和答案泄漏。

## 已有功能

- 按天分配题量；自由加练 30/40/50 题
- 随机、模块、套题层级筛选和搜索
- 上一题、下一题、答题卡、交卷结果
- 错题本、收藏、最近练习
- “不会 / 模糊 / 认识”三档间隔背题
- 不限时持续计时的 30 题模拟考试
- 白板草稿纸和四则运算计算器
- IndexedDB 本地记忆，支持 5 个本地用户档案
- 全用户进度 JSON 导出/导入
- Service Worker 离线外壳和按需题库缓存

## 离线范围

- 首次联网打开后，应用外壳和题库目录会在安装阶段缓存；已经打开的数据分片按需缓存。
- 已经加载过的当前练习可以断网刷新并继续，IndexedDB 中的本机进度不受影响。
- 尚未加载的数据分片不能在断网时临时抽取，因此当前不是“整套题库一次下载后永久离线”的模式。
- 手机端使用 `touch-action: manipulation` 抑制双击放大造成的布局跳动，同时保留正常滚动和双指缩放。

## 验证与发布

```bash
node scripts/validate-project.mjs
node --check app.js
```

GitHub Actions 会先执行以上校验，成功后才部署 GitHub Pages。任何题目只要存在空选项、答案不对应、答案外露、目录/跨题污染、资源缺失或隔离状态不一致，发布都会失败。
